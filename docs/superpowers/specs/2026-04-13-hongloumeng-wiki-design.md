# 红学知识库 LLM Wiki 改造设计

**日期**: 2026-04-13
**状态**: 已批准
**负责人**: 红学虾

---

## 背景

参考 LLM Wiki 模式（Anthropic Patterns: LLM Wiki），将红学研究知识库从传统的文档存储改造为三层架构的维基系统，使 LLM 能够主动维护、增量更新、交叉引用的知识库。

**核心原则**:
- 个人记忆系统（MEMORY.md / episodic / procedural）保持不变
- 只改造红学研究专项知识库
- 红学虾主动搜索发现为主，偶尔提供链接

---

## 一、整体架构（三层）

```
Raw Sources（原始文献）
    ↓  LLM 读取
Wiki（LLM 维护的 markdown 页面）
    ↓  LLM 读写
Schema（WIKI_SCHEMA.md — 规范文档）
```

### 层级说明

| 层级 | 说明 | LLM 权限 |
|------|------|---------|
| Raw Sources | 原始文献（不可修改）| 只读 |
| Wiki | LLM 生成维护的页面 | 读写 |
| Schema | 规范文档 | 读写 |

## 四、Ingest 流程（摄入新文献）

当红学虾搜索到一篇新文章时：

1. **保存原始文献**：保存到 `raw/` 目录，文件名格式 `YYYY-MM-DD-标题.md`
2. **读取并提炼**：红学虾读取文章，提炼关键信息
3. **更新相关页面**：找到对应人物/主题/章节页面，补充新观点、标注矛盾点、更新相关链接
4. **追加 log.md**：`## [date] ingest | 文章标题`
5. **更新 index.md**：如有新页面需求，创建页面

---

## 五、Lint 流程（定期检查）

周期性检查 wiki 健康度：

- **矛盾点检测**：新文献与旧观点冲突时，在页面内标注「争议」
- **孤儿页面**：无交叉引用的页面
- **缺失链接**：提及但未建页面的概念
- **待填内容**：标注为 `TODO` 的空缺内容

## 八、三阶段执行计划

### 阶段一：目录结构
- 创建 `memory/hongloumeng-wiki/` 目录
- 创建 `WIKI_SCHEMA.md`
- 创建空的 `index.md` 和 `log.md`
- 创建 `raw/` 子目录

### 阶段二：迁移
- 迁移现有 `semantic/` 文件到 `hongloumeng-wiki/raw/`
- 分析现有内容，建立初始页面结构（人物/主题/章节/概念各自独立）
- 补充原始文件中缺失的交叉引用

### 阶段三：wiki 工作流
- 建立 `index.md` 和 `log.md` 的维护流程
- 红学虾在日常研究中实践 Ingest/Lint 流程
- 根据实际运行情况调整 Schema

## 九、预期收益

1. **知识累积**：每次研究不再重复「发现」，新文献自动整合入已有框架
2. **交叉引用**：人物/主题/章节页面互相链接，形成网状知识结构
3. **矛盾显性化**：新旧观点冲突时主动标注，避免认知偏差
4. **研究效率**：未来续写时可直接查询 wiki，而非重新检索全文
